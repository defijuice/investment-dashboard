Git 커밋 및 푸시 작업을 수행해주세요.

## 수행 절차

1. **현재 상태 확인**
   - `git status`로 변경된 파일 확인
   - `git diff`로 변경 내용 확인

2. **커밋 메시지 작성**
   - 변경 내용을 분석하여 적절한 커밋 메시지 생성
   - Conventional Commits 형식 사용:
     - `feat:` 새로운 기능
     - `fix:` 버그 수정
     - `docs:` 문서 변경
     - `refactor:` 리팩토링
     - `chore:` 기타 변경
   - 한글로 작성

3. **커밋 및 푸시**
   - 모든 변경 파일 스테이징 (`git add .`)
   - 커밋 생성
   - 원격 저장소에 푸시

4. **결과 보고**
   - 커밋된 파일 목록
   - 커밋 메시지
   - 푸시 결과

## 주의사항

- `.env` 파일이 포함되어 있으면 경고하고 사용자에게 확인 요청
- 커밋 전 사용자에게 커밋 메시지 확인 요청
- 에러 발생 시 원인과 해결 방법 안내
