# 출력 패턴

일관된 고품질 출력을 위한 가이드입니다.

## 템플릿 패턴

구조화된 출력이 필요할 때 사용합니다.

### 엄격한 템플릿

YAML frontmatter 등 정확한 형식이 필요한 경우:

```yaml
---
name: skill-name
description: |
  스킬 설명
  - "트리거 예시 1"
  - "트리거 예시 2"
---
```

### 유연한 템플릿

상황에 맞게 조정이 필요한 경우:

```markdown
# [스킬이름]

[기능 설명 - 상황에 맞게 작성]

## 사용법

[사용 예시 - 스킬 특성에 맞게 작성]
```

## 예시 패턴

구체적인 입출력 예시로 기대 결과를 명확히 합니다.

### 커밋 메시지 예시

**입력:**
```
새 스킬 추가함
```

**출력:**
```
feat: 새 스킬 추가

- SKILL.md 작성
- scripts 추가

🤖 Generated with Claude Code
```

### 완료 메시지 예시

**입력:**
스킬 생성 완료 상태

**출력:**
```
🎉 스킬 생성 완료!

📦 저장소: https://github.com/username/skill-name

📥 팀원 설치 명령어:
curl -L https://github.com/username/skill-name/raw/master/skill-name.tar.gz | tar -xz -C .claude/skills/

🚀 사용: 스킬 트리거 조건에 맞는 요청을 하면 자동 실행됩니다.
```

## 진행 상황 표시 패턴

사용자에게 진행 상황을 알립니다.

```
✓ 스킬 구조 생성 완료
✓ SKILL.md 작성 완료
✓ scripts 작성 완료
⏳ 검증 중...
✓ 검증 통과
⏳ 패키징 중...
✓ tar.gz 생성 완료
⏳ GitHub에 푸시 중...
✓ 배포 완료!
```

## 에러 메시지 패턴

문제 상황을 명확히 전달합니다.

```
❌ 에러: SKILL.md를 찾을 수 없습니다

원인: 지정된 경로에 SKILL.md 파일이 존재하지 않습니다.
경로: /path/to/skill

해결 방법:
1. 스킬 경로가 올바른지 확인하세요
2. init_skill.py로 스킬을 초기화하세요
```

## 핵심 원칙

> 예시는 설명보다 스타일과 세부 수준을 더 명확하게 전달합니다.

- 추상적 설명보다 구체적 예시 사용
- 일관된 이모지와 기호 사용
- 다음 단계를 명확히 안내
