# 출자사업 데이터베이스 테이블 구조

## 개요

모태펀드, KVIC 등 출자사업의 접수현황과 선정결과를 추적하기 위한 테이블 구조입니다.

---

## 테이블 1: 운용사

운용사(GP) 마스터 테이블

| 필드명 | 타입 | 설명 |
|--------|------|------|
| ID | Text | Primary key (OP0001, OP0002, ...) |
| 운용사명 | Text | 운용사 정식 명칭 |
| 약어 | Text | 운용사 별칭/약어 |
| 유형 | Single Select | 국내VC, 해외VC, 창투사, 자산운용사 등 |
| 국가 | Single Select | 한국, 미국, 유럽/중동, 아시아 등 |

### 유형 옵션
- 국내VC
- 해외VC
- 창투사
- 자산운용사
- 기타

### 국가 옵션
- 한국
- 미국
- 유럽/중동
- 아시아
- 기타

## 테이블 2: 출자사업

출자사업 공고 마스터 테이블

| 필드명 | 타입 | 설명 |
|--------|------|------|
| ID | Text | Primary key (PJ0001, PJ0002, ...) |
| 사업명 | Text | 출자사업 공고명 |
| 소관 | Single Select | 주관 기관/계정 |
| 공고유형 | Single Select | 정시, 수시 |
| 연도 | Number | 2025 등 |
| 차수 | Text | 1차, 2차, 8월, 9월, 하반기 등 |
| 지원파일ID | Text | 접수현황 PDF 파일 ID 참조 (쉼표로 연결) |
| 결과파일ID | Text | 선정결과 PDF 파일 ID 참조 (쉼표로 연결) |

### 소관 옵션
- 중기부
- 문화계정
- 문화,관광
- 특허청
- KVIC(해외VC)
- 부산시
- 기타

### 공고유형 옵션
- 정시
- 수시

### 사업명 예시
- 한국모태펀드(중기부 소관) 2025년 2차 정시
- 모태펀드(문화계정) 2025년 9월 수시
- 모태펀드(문화, 관광) 2025년 8월 수시
- 부산 혁신 스케일업 벤처펀드 2025년
- LP 첫걸음펀드 출자사업
- 해외VC 글로벌 펀드 2025년 하반기

## 데이터 입력 흐름

### 1. 접수현황 발표 시
1. 출자사업 테이블에 해당 공고 생성 (없으면)
2. 운용사 테이블에 새 운용사 추가 (없으면)
3. 신청현황 테이블에 레코드 생성
   - 상태: **접수**
   - 공동GP인 경우: 운용사 필드에 여러 운용사 링크

### 2. 선정결과 발표 시
1. 해당 출자사업의 신청현황 레코드들 조회
2. 선정된 운용사: 상태를 **선정**으로 변경, 선정발표일 입력
3. 나머지 운용사: 상태를 **탈락**으로 변경

---

## 활용 예시

### 운용사별 선정률 조회
- 운용사 테이블 → 신청내역 역참조
- 상태별 카운트로 접수/선정/탈락 건수 확인

### 특정 공고 경쟁률 조회
- 출자사업 테이블 → 신청내역 역참조
- 신청 건수 vs 선정 건수

### 출자분야별 트렌드 분석
- 신청현황 테이블 그룹화
- 연도별, 분야별 결성예정액/출자요청액 합계

## 테이블 4: 파일

PDF 파일 처리 이력 관리 테이블

| 필드명 | 타입 | 설명 |
|--------|------|------|
| ID | Text | Primary key (FH0001, FH0002, ...) |
| 파일명 | Text | 원본 파일명 (예: 4524_KVIC_2025_선정결과.pdf) |
| 파일번호 | Text | 파일 prefix (예: 4524) |
| 파일유형 | Single Select | 접수현황, 선정결과 |
| 파일URL | Text | Google Drive 파일 URL |
| 처리상태 | Single Select | 대기, 처리중, 완료, 오류 |
| 처리일시 | DateTime | 마지막 처리 시점 |
| 비고 | Text | 오류 메시지, 처리 결과 등 |

### 파일유형 옵션

- 접수현황
- 선정결과

### 처리상태 옵션

- 대기: 아직 처리 안 됨
- 처리중: 현재 처리 중
- 완료: 정상 처리됨
- 오류: 처리 중 에러 발생

### 활용

- 중복 처리 방지: 이미 완료된 파일 재처리 전 확인
- 처리 이력 추적: 언제 어떤 파일이 처리됐는지 확인
- 역추적: 신청현황 레코드가 어떤 파일에서 생성됐는지 확인 (신청현황.파일ID 참조)
- 오류 관리: 실패한 파일 확인 및 재처리
